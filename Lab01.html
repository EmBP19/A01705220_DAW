Laboratorio 1

  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    • El internet es la red de computadoras conectadas que utilizas para acceder a la world wide web. La web son las paginas que ves gracias al Internet.

  • ¿Cuáles son las partes de un URL?

    • Esquema: le dice al servidor web cual protocolo usar al acceder a la pagina.
    Ej: http:://

    Sub-dominio: indica a que pagina particular de tu sitio el navegador debe mostrar.
    Ej: http:://blog.xxxxxxxx.xxx

    Dominio de segundo nivel: se trata del nombre de tu sitio, identifica la pagina visitada
    Ej: http:://blog.emmanuel.xxx

    Dominio de alto nivel: indica a que tipo de entidad se identifica tu organizacion en internet.
    Ej: http:://blog.emmanuel.com

    Subdirectorio: ayuda a entender en que seccion te encuentras del sitio.
    Ej: http:://blog.emmanuel.com/ventas

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • GET: se utiliza para recuperar datos.
    HEAD: igual a una peticion GET, pero sin el cuerpo de respuesta.
    POST: envia una entidad a un recurso especifico.
    PUT: reemplaza las representaciones actuales con la carga de la peticion.

    PATCH: realiza un cambio la localicación de un recurso.
    DELETE: borra un recurso.

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    • Utilizaria el metodo POST, para mantener la seguridad del usuario.

  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    • Utiliza el metodo GET.

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    • Significa que la peticion se ha realizado con exito.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    • No. Si existe un eror 404, significa que el servidor no encontro el sitio pedido. Esto puede significar que la pagina ha sido borrada, o que el url es incorrecto, o que el sitio se movió. Pero todos estos errores son de parte del cliente.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    • Si. El error esta en la parte del servidor, por lo que debe resolverse de manera interna.

  • Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

    • Utilizarlo solo cuando sea necesario, por ejemplo, que solo arreglas el problema que tienes con eso. O cuando sabes perfectamnte que es lo que haces y que no afectará nada mas.

  • Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

    • Por la resolucion de la imagen, y por consecuente, las pantallas en donde se estara viendo. Por la accesibilidad al usuario, se deben considerar si es facil de leer el contenido.

  • Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

    • Creo que cada una tiene su lugar para utilizarse excepto px. Quizas solo para imagenes o bordes. En cuanto a % y pt, yo usaria el porcentage para pocisionar algo relativo a otro elemento o la pantalla, y pt para algo estatico y texto.

  • ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?
